HTTP协议请求结构

HTTP协议

  • HTTP: Hypertext Transfer Protocol
  • 作用: 是一种用于传输超文本的协议。
  • 是客户端和服务器之间进行通信的基础，常用于在Web浏览器和Web服务器之间传输数据。
  • 客户端与服务端之间通过交换一个个独立的消息进行通信, 客户端发出的称为请求, 服务端发出的应答消息称为响应

HTTP请求的基本结构

示例:

POST /api/login HTTP/1.1 
Host: example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36
Content-Type: application/json

{"username": "user123", "password": "pass456"}
  • 请求行（Request Line）
  • 请求头（Header）
  • 请求体（Body）

请求行

  • 请求行位于请求报文的开头(示例第一行), 包括:

    • 请求方法
    • URL
    • HTTP版本

  • 请求方法:

请求方法 描述 用途
GET 请求获取指定资源 获取网页、图片、文本等资源
POST 在服务器上创建新的资源 提交表单数据、上传文件等
PUT 更新服务器上的资源 更新已存在的资源
DELETE 请求服务器删除指定资源 删除资源
HEAD 请求获取响应头信息 获取资源的头部信息，而不获取资源本身
OPTIONS 请求查看支持的方法 查询服务器支持的HTTP方法
PATCH 对资源进行部分修改 部分更新资源

请求头

  • HTTP请求头提供了关于请求, 响应或者其他的发送实体的信息

  • 示例中:

    • Host：指定服务器的域名和端口号。

    • User-Agent：浏览器或客户端的信息。

    • Content-Type：请求体的数据类型，如application/json。

  • 其他常见请求头，如Accept、Authorization等, 手册:HTTP 标头（header） - HTTP | MDN (mozilla.org)

请求体

  • 可选, 主要用于再请求中携带一些数据(表单等)
  • 主要用于POST, PUT等需要向服务器提交数据的请求
  • Content-Type请求头用于指示请求体的媒体类型, 例如:
    • application/x-www-form-urlencoded：用于普通表单数据。
    • multipart/form-data：用于文件上传等表单数据。
    • application/json：用于JSON数据。(示例)
    • text/plain：用于纯文本数据等。
  • Content-Type手册: HTTP content-type

状态码

  • 当访问一个网页时, 客户端向服务端发出请求, 服务端返回一个包含HTTP状态码的信息头(server header)

  • 常见的有:

    • 200 - 请求成功

    • 301 - 资源（网页等）被永久转移到其它URL

    • 404 - 请求的资源（网页等）不存在

    • 500 - 内部服务器错误

  • 状态码分类:

分类 分类描述
1xx 信息，服务器收到请求，需要请求者继续执行操作
2xx 成功，操作被成功接收并处理
3xx 重定向，需要进一步的操作以完成请求
4xx 客户端错误，请求包含语法错误或无法完成请求
5xx 服务器错误，服务器在处理请求的过程中发生了错误
  • 参考手册: HTTP 响应状态码 - HTTP | MDN (mozilla.org)
